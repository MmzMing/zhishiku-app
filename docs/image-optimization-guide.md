# 图片优化指南

## 为什么需要图片优化

1. **提高页面加载速度**：优化后的图片体积更小，加载更快
2. **改善用户体验**：减少等待时间，降低跳出率
3. **节省带宽**：对用户和服务器都有好处
4. **SEO友好**：页面加载速度是搜索排名的一个因素

## 图片优化策略

### 1. 选择合适的图片格式

- **JPEG/JPG**：适用于照片和复杂图像，有损耗压缩
- **PNG**：适用于需要透明背景或文本/线条图形
- **WebP**：现代格式，提供更好的压缩比，支持透明
- **AVIF**：最新格式，压缩效率更高
- **SVG**：矢量图，适合图标和简单图形

### 2. 压缩图片

- 使用工具如 TinyPNG、ImageOptim 或 Squoosh
- 在线工具：https://tinypng.com/、https://squoosh.app/
- 保持视觉质量的同时减小文件体积

### 3. 响应式图片

- 使用 `srcset` 和 `sizes` 属性提供不同尺寸的图片
- 根据设备和屏幕尺寸加载合适的图片

### 4. 图片懒加载

- 只加载用户视口附近的图片
- 已在项目中通过 v-lazy 指令实现

### 5. CDN加速

- 使用CDN分发图片资源
- 利用CDN的边缘缓存

## 项目中的应用

1. **public/picture 目录**：
   - 所有图片应压缩后上传
   - 预览图 preview.jpg 太大(3.8MB)，建议压缩至100KB以下
   - 考虑使用WebP格式替代JPG/PNG

2. **Mock数据中的图片**：
   - 当前使用的是 picsum.photos 的随机图片
   - 建议使用更有针对性的占位图服务

3. **头像和小图标**：
   - 尺寸应严格控制
   - 推荐使用SVG格式的图标
